# Fix 404 on southernoptimisation.com

Your bucket listing looks **correct** (index.html + assets/ + images/ at the top level).

The error `NoSuchKey: index.html` usually means either:
- **Static website hosting is not enabled**, or
- **Files are not publicly readable**, or
- **Cloudflare** (your domain uses Cloudflare) is pointing at the wrong S3 URL.

---

## Step 1 — Confirm folders are not empty

In bucket **southernoptimisation.com**, click into each folder:

| Folder | Must contain |
|--------|----------------|
| **assets/** | 2 files: `index-BH0tto17.js`, `index-BnQh0tZy.css` |
| **images/** | 10 files (arc-*.png, game-*.jpg, arc-hero.png, logos, etc.) |

If **images/** or **assets/** is empty, re-upload from:

`Desktop/Tempest Cheats/s3-upload`

---

## Step 2 — Enable static website hosting

1. Bucket **southernoptimisation.com** → **Properties**
2. **Static website hosting** → **Edit**
3. **Enable**
4. Hosting type: **Host a static website**
5. Index document: `index.html`
6. Error document: `index.html`
7. **Save changes**
8. Copy the **Bucket website endpoint** — use this exact URL to test:

```
http://southernoptimisation.com.s3-website-ap-southeast-2.amazonaws.com
```

Do **not** use `https://southernoptimisation.com.s3.amazonaws.com/...` (that is the REST API, not the website).

---

## Step 3 — Allow public read (fixes 403 / 404)

### A) Block public access

**Permissions** → **Block public access** → **Edit** → **uncheck all 4** → confirm.

### B) Bucket policy

**Permissions** → **Bucket policy** → **Edit** → paste:

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "PublicReadGetObject",
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::southernoptimisation.com/*"
  }]
}
```

**Save**.

---

## Step 4 — Test in this order

1. Open (copy/paste in browser):

   `http://southernoptimisation.com.s3-website-ap-southeast-2.amazonaws.com`

   If this works → S3 is fine; only DNS/Cloudflare needs fixing.

2. Then test your domain:

   `http://southernoptimisation.com`

---

## Step 5 — Fix Cloudflare (your domain uses Cloudflare)

In **Cloudflare** → **DNS** for southernoptimisation.com:

**Option A — Simple (HTTP to S3 website)**

| Type | Name | Target | Proxy |
|------|------|--------|-------|
| CNAME | `@` | `southernoptimisation.com.s3-website-ap-southeast-2.amazonaws.com` | DNS only (grey cloud) |

**Important:** Turn proxy **OFF** (grey cloud) first while testing. Orange cloud + S3 often causes 404.

**Option B — Better (HTTPS)**

Use **CloudFront** in front of S3, then point Cloudflare CNAME to the CloudFront domain.

---

## Quick diagnosis

| URL | Result | Meaning |
|-----|--------|---------|
| Website endpoint works, domain fails | Cloudflare/DNS wrong |
| Both 404 | Hosting off or policy missing |
| 403 on s3.amazonaws.com URL | File exists but not public — fix policy |
| Blank page, not 404 | assets/ or images/ empty |
